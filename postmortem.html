Overall Architecture

The Student Echo app follows a client-server architecture. The client-side is implemented using Android with Java, while the server-side utilizes SQL as the database.

What Went Right

Our team effectively implemented the key features that meakes Student Echo stand out as the premier sstident rating app.

What Went Wrong

Despite careful planning, our team encountered challenges with implementing some more user friendly features due to time constraints and technical complexities.

Starting Over

If starting over, our team would allocate more time for prototyping and testing to identify and address potential issues early in the development process.

Project Size and Composition

The project consists of a team of 5 developers, divided into frontend and backend development teams. The codebase includes over 20,000 lines of Java code for the Android app.

Time Investment

The project timeline spanned 3 months, with an average of 20 hours per week per team member. Additional time was allocated for planning, meetings, and testing.

Design Decisions and Smells

The team opted for a modular design approach to ensure scalability and maintainability. However, some areas of the codebase exhibited code smells, such as tight coupling between components and inconsistent naming conventions.

Outstanding Bugs

As of the project completion, several minor bugs remain unresolved, including occasional crashes when navigating between screens and inconsistent rendering of certain UI elements on older Android devices.

Unexpectedly Successful Features

The bookmarking feature, initially considered a minor addition, proved to be one of the more user friendly parts of our app.

Additional Technologies

Our team did not use any addition tecnologies. Jave and SWL proved to be sufficient for this project. Future iteratins would however introduce some online services such as APIs.

Development Techniques

The team employed Agile methodologies to ensure continuous progress and effective communication among team members.

Project Evolution

Throughout the development process, the project evolved from a basic review submission app to a comprehensive platform with features such as user authentication, bookmarking, and average rating calculation.

Lessons Learned

Key lessons learned include the importance of thorough testing, the value of clear and consistent communication within the team, and the benefits of regular feedback from users.

Conclusions

In conclusion, the Student Echo app represents a successful collaboration effort that resulted in a functional and user-friendly application for reviewing courses and instructors. While challenges were encountered along the way, the team's dedication and adaptability ensured the project's success.
