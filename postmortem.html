Overall Architecture

The Student Echo app adopts a three-tier architecture, ensuring a clear separation between the user interface, business logic, and database management. The front end features intuitive activities for user interactions, while the back end runs on robust Java code ensuring logical operations are efficiently executed. Data is managed through SQL, allowing for reliable and structured data storage and retrieval. This design ensures maintainability, scalability, and a seamless user experience.

What Went Right

The development process saw the successful implementation of most planned features, fostering a positive team environment where collaboration and enjoyment went hand in hand.

What Went Wrong

We faced the common challenge of merge conflicts, underestimating the complexity of our app's architecture and size, which led to a crunch time during system testing.

Starting Over

Looking back, a phased approach starting with a smaller project scope would have been beneficial, alongside integrating test-driven development from the start to avoid last-minute bugs.

Project Size and Composition

The project evolved to include numerous methods and classes with substantial time devoted to system testing, which was unexpectedly time-consuming.

Time Investment

The bulk of our time was invested in system testing, with UI design being the least time-consuming. Surprisingly, the amount of time required for tests was more than anticipated.

Design Decisions and Smells

Our decision to integrate Bookmarks and Action Alerts added significant value, though we did encounter code smells such as tight coupling that we aim to address.

Outstanding Bugs

Minor bugs remain post-development, providing learning opportunities for future projects.

Unexpectedly Successful Features

The Bookmarks and Notification features integrated seamlessly, exceeding our expectations in user engagement and functionality.

Development Techniques

While we did not employ specific methodologies like TDD or pair programming, regular and frequent meetings underpinned our project's progress.

Project Evolution

Our project's direction saw adjustments throughout its iterations, improving feature descriptions and user stories to better meet users' needs.

Lessons Learned

Developing in smaller increments, maintaining fluid communication, and conducting code reviews before merges are practices we aim to incorporate moving forward.

Conclusions

Our experience was a testament to teamwork and adaptability. Despite the pressures, we delivered a comprehensive product, learning valuable lessons about collaboration and project management.
